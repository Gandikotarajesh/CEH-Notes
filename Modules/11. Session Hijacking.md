#### Session
```bash
Session is the time bw the opening and closing of a communication.
```

#### Session Hijacking
```bash
Session Hijacking is where the attacker take over the legitimate TCP communication session.
This can be done by stealing the session ID, Cookies and other like identity theft, information theft, fraud etc, once the authentication from the user is done.
Application-level hijacking involves gaining control over the Hypertext Transfer Protocol (HTTP) user session by obtaining the session IDs.
Network-level hijacking is prevented by packet encryption, which can be achieved with protocols such as IPsec, SSL, and SSH.
```
#### Cookie
```bash
A Cookie is a small piece of data stored on the user's browser by a website. It helps websites remember information about the user (like login status, preferences, or tracking data).
```
#### Session ID
```bash
A Session ID is a unique identifier (usually a random string) assigned to a user when they visit a website. It helps the server track the user's activity during their visit.
```
#### Token
```bash
Token is a digitally signed string used to authenticate a user. Contains user information and permissions. These tokens can be predictable by understanding the patters and brute forcing.
```
#### SPOOFING vs HIJACKING
```bash
Spoofing is when an attacker fakes an identity to trick systems or people into trusting them.
Hijacking is when an attacker takes control of an active session between a user and a server.
```
#### TYPES OF SESSOIN HIJACKING:
```bash
1. Active session hijacking : An attacker finds an active session and takes it over
2. Passive session hijacking : An attacker hijacks a session, and, instead of taking over, monitors and records all the traffic in that session
```
