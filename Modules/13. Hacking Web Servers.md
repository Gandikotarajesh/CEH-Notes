#### Hacking Web Servers
```bash
Web Server is a SW and the HW deceive that is used to host the Website.
HW: Storage (htms, css, js etc) files.
SW: Apache, Microsoft IIS for hosting.
Using HTTP or/and HTTPS allow web-based clients to connect to them and view and download files.
These sites are going to be hosted using public IP addresses.

Client : Users
Webserver : Website is hosted
Database : User data is stored in Database not in web server
```
#### List of common web servers
```bash
1. Apache Server
2. **IIS** (Internet information service) - MSoft
3. Apache tomcat - default in linux OS
4. Ngnix
5. lighttpd
```

#### HTTP Status Code
```bash
1XX -> Informational
2XX -> Success
3XX -> Redirectional
4XX -> Client error
5XX -> Server error
```

#### Footprint a Web Server using Netcat and Telnet
```bin
Netcat (nc) is a versatile TCP/UDP tool for debugging, transfers, and port scanning, while Telnet is an insecure remote login protocol (port 23) mainly used for testing services like HTTP/SMTP. (Use SSH instead, for security.)

nc -vv www.moviescope.com 80.
telnet <target> 
```

```bash
#If working with netcat
 nc -vv www.moviescope.com 80
#hit Enter, the netcat will display the hosting information of the provided domain.
#Now, type GET / HTTP/1.0 and press Enter twice.
#Netcat will perform the banner grabbing and gather information such as content type, last modified date, accept ranges, ETag, and server information.

#If working with telnet

telnet www.moviescope.com 80
#Type GET / HTTP/1.0 and press Enter twice. Telnet will perform the banner grabbing and gather information such as content type, last modified date, accept ranges, ETag, and server information.
```
