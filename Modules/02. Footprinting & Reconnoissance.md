#### FQDN
```bash
Extensive = full port + service detection  OS detection + script scan
nmap -A -p- <target_IP_range>
nmap -A -p 88,389,53,445 <target_IP_range>
# You will be looking for Domain Controller services:
Port 88/tcp (Kerberos)
Port 389/tcp (LDAP)
Port 53/tcp/udp (DNS)
Port 445/tcp (SMB)

Nmap -p389 -sV <target_list>
389 -> LDAP
FQDN = Host.Domain
FQDN = DC.pentester.com

Tip : • Open port 88 + 389 + 445 together → High chance it's a Domain Controller.

# 2nd method
-> Scan ip range
nmap -p 88,135,139,389,445,636,3268,3269 --open <IP-range> -> nmap -p 88,389,445 --open 192.168.1.0/24
-> Query LDAP rootDSE to extract domain info
nmap -p 389 --script ldap-rootdse <IP>

# 3rd method
 Use Reverse DNS Lookup to Find FQDN

or
```

#### FQDN 2.0
```bash
# Scan the network for DC ports (LDAP/Kerberos/SMB)
nmap -p 88,389,445 172.20.0.0/24

# The machine with these ports open is likely the Domain Controller
{
#Ask the DC its FQDN (via LDAP)
ldapsearch -x H ldap://DCIP -s base dnsHostName
# Output will look like:
dnsHostName: DC01.corp.local
} or
nmap -p389 --script ldap-rootdse 172.20.0.16
or 
nmap -p445 --script smb-os-discovery 172.20.0.16
```
#### Find Domain Controller
```bash
Check for 88/TCP kerberos-sec and port 389/TCP LDAP opened which confirms that it is DC
Once Target is Found, Perform the Aggressive scan to get full details about the DC
```
#### Domain controller IP
```bash
Assume that 10.10.1.22 is a machine
nmap shows that host 10.10.1.22 has port 88/TCP kerberos-sec and port 389/TCP LDAP opened which confirms that our DC IP address is 10.10.1.22.
And to find the FQDN (Fully Qualified Domain Name)
`nmap -A -T5 10.10.1.22` → Gives Product version, FQDN and many other.
```
